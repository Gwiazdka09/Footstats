# Professional Maintenance Prompt

Użyj tego promptu w przyszłości, aby utrzymać projekt w najwyższym standardzie dla rekruterów:

---

"Zrób przegląd projektu pod kątem standardów profesjonalnych (Recruiter-Ready). Wykonaj następujące kroki:

1. **Clean Root**: Sprawdź, czy w katalogu głównym nie ma zbędnych skryptów (.py, .bat, .sh) ani plików tymczasowych/debugowych. Jeśli są, przenieś je odpowiednio do `scripts/`, `src/` lub `assets/`.
2. **Standardy Python**: Upewnij się, że pliki znajdują się w folderze `src/` i przestrzegają konwencji package structure.
3. **Update Documentation**:
   - Zaktualizuj `STATUS.md` – upewnij się, że sekcja 'RECENT MILESTONES' odzwierciedla ostatnie zmiany, a metryki (liczba testów, win-rate) są aktualne.
   - Odśwież `README.md` – sprawdź, czy sekcja '📦 Struktura Projektu' jest zgodna z rzeczywistością. Dodaj nowo dodane funkcjonalności do '🌟 Główne Funkcje'.
4. **Wow Factor**: Jeśli dodałeś nową wizualizację, dashboard lub moduł AI, upewnij się, że jest on wyraźnie opisany w README jako 'Key Highlight' dla rekrutera.
5. **Quality Check**: Uruchom testy i zaktualizuj ich status w dokumentacji. Upewnij się, że nie ma 'martwego kodu' (commented out code)."
